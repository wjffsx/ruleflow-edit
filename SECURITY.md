# Security Policy

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously, especially given the industrial IoT (IIoT) and virtual power plant (VPP) contexts this project serves.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via:

1. **GitHub Security Advisory** — Use the [private vulnerability reporting](../../security/advisories/new) feature
2. **Email** — Send details to the project maintainers

### What to Include

- Type of vulnerability (e.g., XSS, prototype pollution, dependency vulnerability)
- Full paths of source file(s) related to the vulnerability
- Steps to reproduce or proof-of-concept
- Potential impact

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Resolution**: Depends on severity — critical issues will be prioritized

### Disclosure Policy

We follow responsible disclosure. Please allow us a reasonable amount of time to address the vulnerability before public disclosure.
