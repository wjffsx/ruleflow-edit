# RuleFlow-Edit 用户操作手册

> **目标读者**: VPPTU 平台规则链配置工程师 / 运维 / 嵌入式设备集成方
> **版本**: v0.3.0（2026-06-24）
> **适用环境**: 浏览器（PC/嵌入式 Linux WebView）

---

## 1. 概述

RuleFlow-Edit 是 VPPTU 平台的可视化规则链编辑器，基于 Preact + LogicFlow 构建，支持：

- **拖拽式节点编排**：80+ 节点类型（输入/过滤/动作/逻辑门/聚合等）
- **可视化连线**：关系类型选择（next/and/or/not/seq...）
- **属性面板内联编辑**：节点参数、运行参数、连接关系
- **节点搜索与命令面板**：Ctrl+F 搜节点、Ctrl+K 命令面板
- **调试模式**：模拟执行、节点状态可视化、消息流
- **嵌入式适配**：自动检测低端 GPU、ARM 设备，禁用重动画
- **导入/导出**：JSON / YAML 规则链文档

界面分两种模式：
- **edit（编辑模式）**：完整功能
- **view（只读模式）**：仅查看与运行（嵌入式 LCD 推荐）

## 4. 节点类型

节点分 5 大类（共 80+），侧栏搜索 `搜索组件...` 输入名称快速定位：

| 类别 | 节点示例 | nodeType 值 | 用途 |
|------|---------|-------------|------|
| **端口** | 输入端口 / 输出端口 | `input_port` / `output_port` | 数据流入/流出点 |
| **过滤条件** | 设备类型、设备ID、测点名称、值范围、越限、时间窗口、位掩码、变化量阈值... | `condition`（按子类型区分） | 判定数据特征 |
| **动作** | 值变换、重命名、标签、丢弃、路由、告警通知、存储写入、聚合写入... | `action` | 加工或副作用 |
| **逻辑门** | AND 条件组 / OR 条件组 / NOT 条件组 | `condition`（含 logicType 字段） | 复合条件 |
| **复合业务** | SOC 监控、需量计算、爬坡速率、分时电价、天气查询、聚合器、调度控制... | `action` / `condition` | 领域逻辑 |

## 5. 输入端口 / 输出端口 — 数据点配置

> 这是最常被问到的节点类型，下面重点说明。

### 5.1 输入端口（rf-input-port）

代表"外部数据流入"的入口，必须**绑定具体数据点**才能在运行时有数据。

**两种配置方式**：

#### 方式 A：编辑器属性面板（推荐）

1. 从左侧栏拖拽 `输入端口` 到画布
2. 单击该节点 → 右侧 `属性` 面板出现
3. 在 `节点名称` 输入框输入数据点标识（如 `soc_monitor`、`active_power`）
4. 高级设置（通过 `查看完整属性` 按钮打开 RightPanel 完整编辑）：
   - `nodeType` 固定为 `input_port`
   - `icon` 默认 `→`
   - `priority` 优先级（默认 0）
   - `enabled` 是否启用
   - `summary` 数据点键名（与后端测点名映射）
   - `ruleId` 所属规则 ID
   - `roleInRule` 固定 `input`

> **数据点名称规范**：
> - 文本框内容 = 节点显示名（如"SOC 数据"）
> - `properties.summary` = 实际数据点 ID（`soc_monitor`，与后端 FQN 对应）
> - 两者必须**分别配置**：前者用于画布展示，后者用于运行时数据路由

#### 方式 B：导入 JSON / YAML

```json
{
  "id": "input_soc",
  "type": "rf-input-port",
  "text": "SOC 数据",
  "properties": {
    "nodeType": "input_port",
    "icon": "→",
    "priority": 0,
    "enabled": true,
    "summary": "soc_monitor"
  }
}
```

`summary` 字段即数据点名称，运行时会作为消息的 `source` 或 `subject`。

### 5.2 输出端口（rf-output-port）

代表"规则链产出"的出口，配置方式同上：

```json
{
  "id": "output_dispatch",
  "type": "rf-output-port",
  "text": "调度指令",
  "properties": {
    "nodeType": "output_port",
    "icon": "←",
    "priority": 0,
    "enabled": true,
    "summary": "dispatch_command"
  }
}
```

输出端口的 `summary` 表示**目标数据点**（订阅者 FQN），运行时会把上游动作节点产生的数据发送到这里。

### 5.3 数据流对照表

| 字段 | 含义 | 何时使用 |
|------|------|---------|
| `text` | 画布显示文本 | 视觉识别，可中文 |
| `properties.summary` | 数据点 FQN | 运行时数据路由，**必须与后端测点一致** |
| `properties.priority` | 评估优先级 | 同链路多入口时排序 |
| `properties.enabled` | 是否参与评估 | 临时禁用，不删除 |
| `properties.ruleId` | 所属规则 | 规则分组 |

## 6. 基础操作

### 6.1 创建节点

- **拖拽法**：从左侧栏拖任意节点到画布
- **节点 ID 自动生成**：`{type}_{timestamp}_{6位随机串}`，避免碰撞

### 6.2 编辑节点

- **单击节点**：右侧弹出属性气泡
- **双击节点**：进入文本编辑模式
- **拖动节点**：移动位置
- **删除节点**：选中后按 `Delete`

### 6.3 创建连线

1. 鼠标悬停节点 → 出现 4 个连接点
2. 从源节点连接点拖到目标节点
3. 松开后弹出 **关系类型选择器**：
   - `next`：顺序流转
   - `and`：与条件
   - `or`：或条件
   - `not`：非条件
   - `seq`：序列
   - 自定义类型（编辑模式下）

### 6.4 校验规则（已自动拦截）

- **自环**：A→A 直接拒绝
- **重复连线**：同源同目标的多条边自动保留首条
- **循环依赖**：A→B→A 通过 DFS 检测后拒绝
- **悬空边**：引用不存在节点的边在加载时被过滤

### 6.5 缩放与平移

- **滚轮**：缩放画布
- **空格 + 拖动** 或 **中键拖动**：平移
- **右下角控件**：`−` 缩小 / `+` 放大 / `↺` 重置
- **快捷键**：`Ctrl+0` 重置 / `Ctrl++` 放大 / `Ctrl+-` 缩小

## 9. 节点搜索（Ctrl+F）

在大型规则链（100+ 节点）中快速定位：

- 输入节点名称、节点 ID 或类型
- 结果按模糊匹配排序
- 回车跳转到该节点并居中

> **性能**：节点数 < 50 时使用原生 `includes()` 匹配，≥ 50 时使用 `fuse.js` 模糊索引。

## 10. 调试模式

### 10.1 启动调试

- 工具栏 `▶ 运行` 或 `F5`
- 或命令面板 `run`

### 10.2 节点状态颜色

| 颜色 | 状态 |
|------|------|
| 🟢 绿色发光 | 成功（已通过） |
| 🔴 红色发光 | 失败（条件不满足） |
| 🟣 紫色脉冲 | 正在处理 |
| ⚪ 默认 | 未触发 |

### 10.3 调试面板（右侧 `调试` Tab）

- **步骤指示器**：当前步 / 总步数
- **消息列表**：每步节点的输入/输出/错误（最多保留 500 条，低性能设备 100 条）
- **暂停 / 继续 / 单步**
- **清空消息**

### 10.4 停止调试

- `■ 停止` 按钮或 `Shift+F5`

- 工业 LCD 触控屏

## 14. 错误处理

### 14.1 渲染错误

任何组件抛错都会被 [ErrorBoundary](file:///e:/VPPTU-06/ruleflow-edit/src/components/common/ErrorBoundary.tsx) 捕获，展示友好错误页 + `重新加载` 按钮。

### 14.2 Toast 通知

- `成功 / 错误 / 警告 / 信息` 四种类型
- 右下角堆叠显示
- 5 秒自动消失，可手动关闭

### 14.3 嵌入式降级

| 条件 | 行为 |
|------|------|
| `navigator.hardwareConcurrency ≤ 4` | SVG filter 仅保留最小阴影，禁用 glow 效果 |
| `navigator.platform` 含 `arm` | 同上 + 调试消息上限 100 |
| `prefers-reduced-motion: reduce` | 全部 CSS 过渡时长 → 0ms |
| 扩展插件加载失败 | 回退到无插件模式（功能可用） |

## 15. 常见问题 FAQ

### Q1：节点之间连不上？
- 自环（A→A）：自动拒绝
- 重复连线（A→B 已存在）：保留首条
- 循环依赖：自动检测并拒绝

### Q2：Ctrl+S 没反应？
- 检查 `onSave` 回调是否传入
- 否则仅显示 toast 提示

### Q3：搜索不到节点？
- 节点 ID 和文本都会匹配
- 输入中文/英文/数字均可

### Q4：调试消息不显示？
- 必须先 `▶ 运行`
- 检查节点是否启用（`properties.enabled`）

### Q5：编辑模式性能差？
- 检查节点数量（> 200 建议拆分规则链）
- 在嵌入式设备上用 view 模式
- 浏览器硬件加速是否开启

### Q6：导出的 JSON 怎么用？
- 通过后端 API 导入到 VPPTU
- 或加载到另一个 RuleFlow-Edit 实例

## 16. 开发集成

### 16.1 作为组件嵌入

```tsx
import { RuleFlowEditor } from 'ruleflow-edit'

<RuleFlowEditor
  mode="edit"                 // 'edit' | 'view'
  data={ruleChainData}        // 初始数据
  onSave={(doc) => api.saveRuleChain(doc)}
  onDebugStart={...}
  hotkeyOverrides={{ 'command-palette': 'ctrl+shift+p' }}
  readOnly={false}
/>
```

### 16.2 事件回调

| 回调 | 触发时机 | 签名 |
|------|---------|------|
| `onSave` | Ctrl+S | `(doc: RuleFlowDocument) => void` |
| `onDebugStart` | F5 | `() => void` |
| `onCommandExecute` | 命令面板 | `(cmdId: string) => void` |

### 16.3 自定义属性渲染器

```tsx
<RuleFlowEditor
  propertyRenderer={(node, onChange) => (
    <MyCustomEditor node={node} onChange={onChange} />
  )}
/>
```

## 18. 技术规格

- **包大小**（gzip）：主包 ~120KB，编辑 chunk 按需加载
- **首屏渲染**：edit 模式 < 500ms，view 模式 < 200ms
- **内存基线**：单页 ~30MB，100 节点 ~35MB
- **支持浏览器**：Chrome 90+ / Edge 90+ / Firefox 88+ / 嵌入式 WebView（Chromium 80+）
- **不依赖**：jQuery、@floating-ui/dom（已移除）
